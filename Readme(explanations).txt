Для данного регулярного выражения заводи структуру PrefixSet в которой будем хранить всю необходимую информацию для нахождения префикса x^k минимальной длины. В этой структуре хранится длина x в массиве AllMinLen для каждого 0 <=i <= k. 
    Создаем стек PrefixSet и пробегая по обычному стеку для регулярок кладем результат операций(pushMul, pushAdd, pushPow) 

    pushMul - операция для соединенияя части двух частей слова.
Выполнение этого очевидно. Время работы одной операции O(1), всех O(k).
    pushAdd тоже более менее очевидно. 
Просто складываем два регулярных выражения. Время работы одной операции O(1), всех O(k).
    pushPow же сложнее и занимает больше времени. 
Время работы же pushPow отличается от pushMul. Рер занимает O(k^2)
Так как для каждой степени нужно пересмотреть все подходящие и неподходящиеся слова в результате получившиеся при сложении двух регулярок.

А значит суммарное время работы - n*k^2, где n длина регулярки, а k - степень префикса.
